# i18n Design Spec

## Overview

Add internationalization to `@vx/core-uikit` using `react-i18next` + `i18next`. Core-uikit owns the i18n infrastructure and common translation namespaces (common, errors, forms, auth). Apps register their own feature-specific translations on top.

**Default locales:** English (`en`), Vietnamese (`vi`), Japanese (`ja`)
**Library:** `react-i18next` + `i18next` + `i18next-browser-languagedetector`

## i18n Setup (`setup.ts`)

Initializes `i18next` with:
- `react-i18next` initReactI18next plugin
- `i18next-browser-languagedetector` for auto-detection
- Bundled resources for all 3 locales x 4 namespaces (common, errors, forms, auth)
- Default namespace: `common`
- Fallback locale: `en`
- Detection order: `localStorage` → `navigator` → fallback
- localStorage key: `vx_locale`
- Interpolation: `escapeValue: false` (React already escapes)

```ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import LanguageDetector from "i18next-browser-languagedetector";

// Import all bundled locale files
import enCommon from "./locales/en/common.json";
import enErrors from "./locales/en/errors.json";
import enForms from "./locales/en/forms.json";
import enAuth from "./locales/en/auth.json";
// ... vi and ja imports

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      en: { common: enCommon, errors: enErrors, forms: enForms, auth: enAuth },
      vi: { common: viCommon, errors: viErrors, forms: viForms, auth: viAuth },
      ja: { common: jaCommon, errors: jaErrors, forms: jaForms, auth: jaAuth },
    },
    defaultNS: "common",
    fallbackLng: "en",
    supportedLngs: ["en", "vi", "ja"],
    interpolation: { escapeValue: false },
    detection: {
      order: ["localStorage", "navigator"],
      lookupLocalStorage: "vx_locale",
      caches: ["localStorage"],
    },
  });

export { i18n };
```

## I18nProvider (`provider.tsx`)

Thin wrapper that ensures i18n is initialized before rendering children. Re-exports `I18nextProvider` from `react-i18next`.

```tsx
import React from "react";
import { I18nextProvider } from "react-i18next";
import { i18n } from "./setup";

type I18nProviderProps = {
  children: React.ReactNode;
  defaultLocale?: string;
};

export function I18nProvider({ children, defaultLocale }: I18nProviderProps) {
  if (defaultLocale && i18n.language !== defaultLocale) {
    i18n.changeLanguage(defaultLocale);
  }
  return <I18nextProvider i18n={i18n}>{children}</I18nextProvider>;
}
```

## useLocale Hook (`use-locale.ts`)

```ts
import { useTranslation } from "react-i18next";

type UseLocaleReturn = {
  locale: string;
  setLocale: (locale: string) => Promise<void>;
  supportedLocales: { code: string; label: string }[];
};

export function useLocale(): UseLocaleReturn {
  const { i18n } = useTranslation();

  const supportedLocales = [
    { code: "en", label: "English" },
    { code: "vi", label: "Tiếng Việt" },
    { code: "ja", label: "日本語" },
  ];

  const setLocale = async (locale: string) => {
    await i18n.changeLanguage(locale);
    localStorage.setItem("vx_locale", locale);
  };

  return {
    locale: i18n.language,
    setLocale,
    supportedLocales,
  };
}
```

## registerTranslations Helper (`register.ts`)

Apps call this at startup to add feature-specific translations.

```ts
import { i18n } from "./setup";

export function registerTranslations(
  locale: string,
  namespace: string,
  resources: Record<string, string>
): void {
  i18n.addResourceBundle(locale, namespace, resources, true, true);
}
```

## Component Integration

Existing core-uikit components will use `useTranslation()` internally:

### DataTable
- "No results" → `t("common:noResults")`
- "Page X of Y (Z total)" → `t("common:showing", { from, to, total })`

### FormBuilder
- Validation messages → `t("forms:required")`, `t("forms:invalidEmail")`, etc.
- Save/Cancel buttons → `t("common:save")`, `t("common:cancel")`

### ConfirmDialog
- Confirm/Cancel buttons → `t("common:confirm")`, `t("common:cancel")`
- Delete variant → `t("common:delete")`

### LoginPage
- Labels → `t("auth:email")`, `t("auth:password")`
- Button → `t("auth:signIn")`, `t("auth:signingIn")`
- Links → `t("auth:forgotPassword")`, `t("auth:noAccount")`

### RegisterPage
- Labels → `t("auth:email")`, `t("auth:password")`, `t("auth:confirmPassword")`
- Button → `t("auth:signUp")`, `t("auth:creatingAccount")`

### ForgotPasswordPage
- Labels → `t("auth:email")`
- Button → `t("auth:sendResetLink")`
- Success → `t("auth:resetLinkSent")`

### API Hooks (use-rest-api.ts)
- Error toast messages → `t("errors:networkError")`, `t("errors:serverError")`, etc. based on HTTP status code

## App Integration Pattern

```tsx
// apps/admin/src/main.tsx
import { I18nProvider, registerTranslations } from "@vx/core-uikit/i18n";

import enUsers from "./i18n/locales/en/users.json";
import viUsers from "./i18n/locales/vi/users.json";
import jaUsers from "./i18n/locales/ja/users.json";
import enDashboard from "./i18n/locales/en/dashboard.json";
import viDashboard from "./i18n/locales/vi/dashboard.json";
import jaDashboard from "./i18n/locales/ja/dashboard.json";

registerTranslations("en", "users", enUsers);
registerTranslations("vi", "users", viUsers);
registerTranslations("ja", "users", jaUsers);
registerTranslations("en", "dashboard", enDashboard);
registerTranslations("vi", "dashboard", viDashboard);
registerTranslations("ja", "dashboard", jaDashboard);

// In the root component
<I18nProvider defaultLocale="en">
  <RouterProvider router={router} />
</I18nProvider>
```

Usage in app components:

```tsx
import { useTranslation } from "react-i18next";

function UserListPage() {
  const { t } = useTranslation("users");
  return <h1>{t("title")}</h1>; // "User Management"
}
```

## Dependencies

Add to `@vx/core-uikit`:
- `i18next`
- `react-i18next`
- `i18next-browser-languagedetector`

## Non-Goals

- Server-side locale detection (SPA only)
- Lazy loading of locale files (bundle all 3 locales — small enough)
- RTL layout support
- Locale-specific date/number formatting (use browser Intl API directly if needed)
- Translation management platform integration (Crowdin, Lokalise, etc.)
