# VX Admin Theme — Design Spec

## Overview

A reusable admin dashboard monorepo built with Tailwind CSS, shadcn/ui, TanStack Router, and TanStack Query. The core package (`@vx/core-uikit`) provides shared components, API layer, layouts, and CRUD generators. Multiple admin apps consume the core.

**Platform:** Web only
**Package scope:** `@vx/*`

### Workspace Layout

- **`apps/`** — Each admin app is an independent Vite + React project that depends on `@vx/core-uikit`. To create a new admin app: scaffold a Vite project in `apps/`, add `@vx/core-uikit` as a workspace dependency, and configure routes.
- **`packages/core-uikit/`** — Single shared package with subpath exports (`@vx/core-uikit/components`, `@vx/core-uikit/api`, `@vx/core-uikit/layouts`, etc.)
- **`tooling/`** — Shared configs for TypeScript, ESLint, and Tailwind. All apps and packages extend these.

### Components (`src/components/`)

**Base components** — sourced via `npx shadcn@latest add`:
- button, input, select, textarea, checkbox, switch, label
- card, dialog, popover, tooltip, dropdown-menu
- table, badge, avatar, separator, skeleton
- sheet (mobile sidebar), scroll-area
- toast (notifications via sonner)

**Custom components** built on shadcn primitives:
- `DataTable` — sortable, filterable, paginated table using TanStack Table + shadcn table
- `FormBuilder` — renders forms from a field config array, integrates react-hook-form + zod validation
- `DetailView` — displays entity detail from field config (read-only view)
- `FilterBar` — search input + filter dropdowns, emits filter state
- `ConfirmDialog` — reusable delete/action confirmation dialog

### Layouts (`src/layouts/`)

Based on shadcn `dashboard-01` block:
- `DashboardLayout` — full sidebar + header + main content area
- `Sidebar` — collapsible sidebar with grouped navigation items, app logo, user menu
- `Header` — breadcrumbs, search, user avatar, notifications
- `ContentArea` — scrollable content wrapper with consistent padding

The layout accepts a `navigation` config array to define sidebar items:

```ts
type NavItem = {
  label: string;
  path: string;
  icon: LucideIcon;
  children?: NavItem[];  // nested group
};

type NavigationConfig = {
  main: NavItem[];
  secondary?: NavItem[];  // bottom of sidebar (settings, help)
};
```

### API Layer (`src/api/`)

Four layers, from low-level to high-level:

#### Layer 1: HTTP Client (`http-client.ts`)

- Wraps `fetch` with typed helpers: `get<T>()`, `post<T>()`, `put<T>()`, `del<T>()`
- Base URL from environment config (`VITE_API_BASE_URL`)
- Auto-attaches `Authorization: Bearer <token>` header from stored token
- 401 detection with automatic token refresh flow:
  1. Request fails with 401 + `X-Token-Expired` header
  2. Call refresh endpoint with stored refresh token
  3. Store new access token
  4. Retry original request
  5. If refresh fails, call unauthorized callback (redirect to login)
- Token storage via `localStorage` (access token + refresh token)
- Request/response interceptor pattern for extensibility

#### Layer 2: API Client (`client.ts`)

```ts
const appApi = {
  get<T>(url: string, options?): Promise<T>,
  getAll<T>(url: string, params?: ListParams): Promise<PaginatedResponse<T>>,
  post<T>(url: string, data?: unknown): Promise<T>,
  put<T>(url: string, data?: unknown): Promise<T>,
  patch<T>(url: string, data?: unknown): Promise<T>,
  delete<T>(url: string): Promise<T>,
};
```

- Query params builder: converts `{ page, limit, sort, filters }` → querystring
- `interpolatePath(template, params)`: e.g., `"/users/:id"` + `{ id: "123" }` → `"/users/123"`

#### Layer 3: Generic TanStack Query Hooks (`hooks/use-rest-api.ts`)

```ts
// Paginated list
useRestGetAll<T>({
  entityName: string,
  url: string,
  page?: number,
  limit?: number,
  filters?: Record<string, unknown>,
  sort?: { field: string; order: "asc" | "desc" }[],
  enabled?: boolean,
}) → { data: T[], meta: PaginationMeta, isLoading, refresh }

// Single entity
useRestGetOne<T>({
  entityName: string,
  url: string,
  pathParams?: Record<string, string>,
  enabled?: boolean,
}) → { data: T, isLoading, refresh }

// Create / Update / Delete
useRestMutation<T>({
  entityName: string,
  url: string,
  method: "POST" | "PUT" | "PATCH" | "DELETE",
  notifications?: {
    success?: (result: T) => { title: string; description: string },
    error?: (error: Error) => { title: string; description: string },
  },
}) → { mutate, data, error, isLoading, reset }

// Infinite scroll list
useRestGetAllInfinite<T>({
  entityName: string,
  url: string,
  limit?: number,
  filters?: Record<string, unknown>,
}) → { items: T[], hasMore, loadMore, isLoadingMore }
```

**Query key convention:**
- List: `[entityName, "list", page, limit, filters, sort]`
- Single: `[entityName, id]`
- Infinite: `[entityName, "infinite", url, params]`
- Mutations auto-invalidate `[entityName]` on success

#### Layer 4: Domain Hooks (per app, NOT in core)

Each admin app creates its own domain-specific hooks using the generic ones:

```ts
// apps/admin/src/api/use-users-api.ts
export function useUsers(params) {
  return useRestGetAll<User>({ entityName: "users", url: "/api/users", ...params });
}
export function useCreateUser() {
  return useRestMutation<User>({ entityName: "users", url: "/api/users", method: "POST" });
}
```

### Auth (`src/hooks/` + `src/components/`)

**AuthProvider** — React context wrapping the app:

```ts
useAuth() → {
  user: User | null,
  isAuthenticated: boolean,
  isLoading: boolean,
  login: (credentials) => Promise<void>,
  logout: () => void,
  register: (data) => Promise<void>,
}
```

**Auth flow:**
1. On app load, check localStorage for existing tokens
2. If token exists, call `/auth/me` to validate and get user
3. If expired, attempt token refresh
4. If no valid session, redirect to login

**Protected routes** via TanStack Router `beforeLoad`:

```ts
// In route definition
beforeLoad: ({ context }) => {
  if (!context.auth.isAuthenticated) {
    throw redirect({ to: "/login" });
  }
}
```

**Reusable auth pages in core:** `LoginPage`, `RegisterPage`, `ForgotPasswordPage` — styled with shadcn, configurable (logo, title, fields).

export const endpoints = {

### CRUD Page Generator (`src/generators/`)

#### Field Config

```ts
type FieldConfig = {
  name: string;
  label: string;
  type: "text" | "email" | "number" | "select" | "textarea" | "date" | "boolean" | "relation";
  options?: { label: string; value: string }[];  // for select type
  sortable?: boolean;
  filterable?: boolean;
  showInList?: boolean;    // show in table (default: true)
  showInDetail?: boolean;  // show in detail view (default: true)
  showInForm?: boolean;    // show in create/edit form (default: true)
  validation?: ZodType;    // per-field zod validation
};
```

#### Generator Function

```ts
function createCrudPages<T>(config: {
  entityName: string;
  basePath: string;
  apiUrl: string;
  fields: FieldConfig[];
  // Optional overrides
  columns?: ColumnDef<T>[];      // custom TanStack Table columns
  formSchema?: ZodSchema;         // custom zod schema (overrides field-level)
  listActions?: (row: T) => ReactNode;  // custom row actions
  pageSize?: number;              // default 20
}): {
  ListPage: React.FC;
  CreatePage: React.FC;
  EditPage: React.FC;
  DetailPage: React.FC;
}
```

**What it generates:**
- `ListPage` — DataTable with pagination, sorting, filtering, row actions (view/edit/delete), "Create new" button
- `CreatePage` — FormBuilder with validation, submit → `POST apiUrl`, redirect to list on success
- `EditPage` — FormBuilder pre-filled with existing data, submit → `PUT apiUrl/:id`
- `DetailPage` — DetailView showing all fields, edit/delete action buttons

Apps wire these into their TanStack Router file-based routes.

### Root Layout Pattern

```tsx
// routes/__root.tsx
import { DashboardLayout } from "@vx/core-uikit/layouts";
import { QueryProvider } from "@vx/core-uikit/api";
import { AuthProvider } from "@vx/core-uikit/hooks";

  component: () => (
    <QueryProvider>
      <AuthProvider>

        </DashboardLayout>
      </AuthProvider>
    </QueryProvider>
  ),

```

## Creating a New Admin App

1. Create `apps/my-new-admin/` with Vite React template
2. Add `@vx/core-uikit` as workspace dependency
3. Configure TanStack Router file-based routing
4. Define navigation config for sidebar
5. Define endpoint config for your API
6. Create domain-specific hooks using generic `useRestGetAll`, `useRestMutation`, etc.
7. For standard CRUD: use `createCrudPages()` with field config, wire into routes
8. For custom pages: compose `DataTable`, `FormBuilder`, etc. directly

## Non-Goals (Out of Scope)

- SSR / server components (pure SPA)
- Cross-platform (web only)
- Real-time / WebSocket support
- File upload abstraction (can be added later)
- i18n / localization (can be added later)
- Role-based access control (can be added later)
- State management beyond TanStack Query (no Redux, Zustand, etc.)
