# Detail & Edit UI Redesign — Design Spec

**Date:** 2026-05-12
**Status:** Approved
**Scope:** Custom detail and edit pages for Books and Categories replacing generic CRUD generator output

---

## 1. Overview

Replace the flat key-value detail pages and single-column edit forms with product-style split-panel layouts. Both Books and Categories get custom pages — these are NOT generated by `createCrudPages` but built as standalone components using shared UI primitives from core-uikit.

**Key principle:** Left panel = visual content (sticky), Right panel = information/forms (scrollable).

---

## 2. Book Detail Page (`/books/:bookId`)

### Layout

Two-column split panel. Left sticky at 280px, right scrollable.

### Left Panel (Sticky Media)

| Section            | Content                                                                                 |
| ------------------ | --------------------------------------------------------------------------------------- |
| Cover Image        | Large cover (full width, ~220px height), loaded via `imageBaseUrl + coverUrl`           |
| Thumbnail Variants | 2-column grid: Thumbnail (3:4) + Square Thumbnail (1:1), show image or dash placeholder |
| Specifications     | Card: Pages (number), Dimensions, Age Range — compact key-value                         |
| Pricing            | Card: Price (green), Original Price, Discount — compact key-value                       |
| Files              | Card: PDF Download link + Tryout Page link, show as clickable links or dash if empty    |

### Right Panel (Scrollable Content)

| Section             | Content                                                                                                                                       |
| ------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| Book Information    | Card: Title, Subtitle, Category (text), Badge (colored chip), Background Color (swatch + hex). Description as paragraph below separator       |
| Coloring Pages      | Card: Title with page count. 5-column image grid. Green dot on public pages. "+N more" tile if >8 pages shown initially. Click opens lightbox |
| Summary Pages       | Card: Same grid layout as coloring pages                                                                                                      |
| Settings & Metadata | Card: Converted/Redesigned/Edition Converted as checkmark labels. Created/Updated dates                                                       |

### Header Bar

- Back button (left)
- Book title + badge chip (center-left)
- Edit button + Delete button (right)

### Lightbox (on thumbnail click)

- Dark overlay, full-viewport
- Large image centered with prev/next arrows
- Bottom metadata bar: "Page N of M" • filename • ID • Public indicator (green dot + label) • Close button
- Navigate through pages with arrow keys or click

## 3. Category Detail Page (`/categories/:categoryId`)

### Layout

Same two-column split. Left 280px sticky, right scrollable.

### Left Panel

| Section     | Content                                                             |
| ----------- | ------------------------------------------------------------------- |
| Icon        | Large icon image (180px height), centered in white card             |
| Quick Stats | Card: Sort Order, Book count, Visibility (Public/Hidden with color) |
| Icon Prompt | Card: The AI prompt text in italic, smaller font                    |

### Right Panel

| Section              | Content                                                                                                                       |
| -------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| Category Information | Card: Display Name, Internal Name (monospace), Description as paragraph                                                       |
| Books in Category    | Card: Title with book count. 4-column card grid, each card = cover image + title + price. Click navigates to `/books/:bookId` |

### Header Bar

Same pattern as books: Back, Category name + Public/Hidden badge, Edit + Delete buttons.

## 4. Book Edit Page (`/books/:bookId/edit`)

### Layout

Same two-column split. Left = read-only media preview, Right = form fields.

### Left Panel (Read-Only Preview)

- Cover image preview (updates live if cover URL field changes)
- Thumbnail variant previews (3:4 + 1:1)
- Coloring Pages mini-grid (3 cols, compact, shows first 5 + count)
- Summary Pages mini-grid

This panel is NOT editable — it reflects the current saved state (or live form state for URL fields).

### Right Panel (Form Cards)

| Card                      | Fields                                                                                                                                                               |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Basic Information         | Title\*, Subtitle, Description (textarea), Category (dropdown), Badge (dropdown)                                                                                     |
| Pricing                   | Price, Original Price, Discount — 3 columns                                                                                                                          |
| Image URLs                | Cover URL\*, Thumbnail URL, Square Thumbnail, PDF URL, Tryout Page URL — with inline preview                                                                         |
| Specifications + Settings | Side-by-side 2 columns. Left: Pages (number), Dimensions, Age Range. Right: Converted/Redesigned/EditionConverted (toggle switches), Background Color (color picker) |

### Header Bar

- Cancel button (left, goes back)
- "Edit: [Book Title]" (center-left)
- Save button (right, primary blue)

### Coloring/Summary Pages Editing

Coloring Pages and Summary Pages arrays are edited via the existing `nested-array-field` component in a separate section below the main form, or remain on the detail page only (not editable from the edit page — these are typically managed via separate tooling).

**Decision:** Keep coloring/summary page arrays as **view-only on edit page** (shown in left preview panel). Editing these arrays of 50+ items through form fields is impractical. If editing is needed later, a dedicated page/modal can be added.

## 5. Category Edit Page (`/categories/:categoryId/edit`)

### Layout

Same split. Left = icon preview, Right = form.

### Left Panel

- Icon image preview (large)
- Current book count (read-only stat)

### Right Panel

| Card              | Fields                                                                                                |
| ----------------- | ----------------------------------------------------------------------------------------------------- |
| Basic Information | Display Name*, Internal Name*, Description (textarea)                                                 |
| Settings          | Icon URL (url-image field with preview), Icon Prompt (textarea), Sort Order (number), Public (toggle) |

Books array is read-only (managed from book side via categoryId).

## 6. Implementation Approach

These are **custom page components** in `apps/admin/src/pages/`, NOT generated by the CRUD generator. They import:

- Shared UI from `@vx/core-uikit/components` (Card, Badge, Button, etc.)
- Firestore hooks from `@vx/core-uikit/firebase` (useFirestoreGetOne, useFirestoreMutation)
- Form handling from `react-hook-form` + `zod`
- `imageBaseUrl` from env for resolving relative URLs

### New Files

| File                                            | Purpose                                    |
| ----------------------------------------------- | ------------------------------------------ |
| `apps/admin/src/pages/book-detail-page.tsx`     | Custom book detail with split panel        |
| `apps/admin/src/pages/book-edit-page.tsx`       | Custom book edit form with preview panel   |
| `apps/admin/src/pages/category-detail-page.tsx` | Custom category detail with split panel    |
| `apps/admin/src/pages/category-edit-page.tsx`   | Custom category edit form                  |
| `apps/admin/src/components/image-lightbox.tsx`  | Lightbox modal for page thumbnails         |
| `apps/admin/src/components/image-grid.tsx`      | Reusable image grid with public indicators |
| `apps/admin/src/components/detail-card.tsx`     | Reusable card wrapper for detail sections  |

### Modified Files

| File                                                     | Change                             |
| -------------------------------------------------------- | ---------------------------------- |
| `apps/admin/src/routes/books/$bookId.tsx`                | Point to custom BookDetailPage     |
| `apps/admin/src/routes/books/$bookId_.edit.tsx`          | Point to custom BookEditPage       |
| `apps/admin/src/routes/categories/$categoryId.tsx`       | Point to custom CategoryDetailPage |
| `apps/admin/src/routes/categories/$categoryId_.edit.tsx` | Point to custom CategoryEditPage   |

### NOT Changed

- Book/Category list pages — stay as CRUD generator output (working fine)
- Create pages — stay as CRUD generator (same form, no preview needed)
- Other entities (wallets, etc.) — keep generic CRUD

## 7. Shared Components

### ImageGrid

```tsx
<ImageGrid
  images={coloringPages}
  imageBaseUrl={imageBaseUrl}
  columns={5}
  maxVisible={8}
  showPublicIndicator
  onImageClick={(page, index) => openLightbox(index)}
/>
```

### ImageLightbox

```tsx
<ImageLightbox
  images={coloringPages}
  imageBaseUrl={imageBaseUrl}
  initialIndex={clickedIndex}
  open={isOpen}
  onClose={() => setOpen(false)}
/>
```

Shows: large image, prev/next navigation, bottom bar with page ID, URL, public status.

### DetailCard

```tsx
<DetailCard title="Book Information" className="...">
  {children}
</DetailCard>
```

Simple wrapper: Card with title header and content body.
